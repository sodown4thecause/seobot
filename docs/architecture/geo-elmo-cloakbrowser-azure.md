# GEO / AEO Engine: Elmo + CloakBrowser on Azure

**Status:** Approved direction  
**Last updated:** 2026-06-09  
**Implementation repository:** [sodown4thecause/geomode](https://github.com/sodown4thecause/geomode)  
**Upstreams:** [elmohq/elmo](https://github.com/elmohq/elmo) and [CloakHQ/CloakBrowser](https://github.com/CloakHQ/CloakBrowser)

## Decision

FlowIntent's GEO / AEO mode will use the
[`sodown4thecause/geomode`](https://github.com/sodown4thecause/geomode) fork of
Elmo as its prompt scheduling, capture, citation, and visibility-analysis
engine.

The fork will add a `cloakbrowser` provider to Elmo's existing provider registry.
It replaces Bright Data for browser-based ChatGPT, Perplexity, and Google AI
surface capture. The existing FlowIntent app remains the customer-facing chat,
artifact, workspace, billing, and authorization layer.

The browser engine runs on Azure, not in Vercel request handlers.

## Runtime Architecture

```text
FlowIntent (Vercel)
  |
  | authenticated GEO run request / result sync
  v
geomode / Elmo fork (Azure Docker stack)
  |- web/API
  |- pg-boss worker
  |- PostgreSQL
  `- CloakBrowser provider
       |- ChatGPT
       |- Perplexity
       `- Google AI Mode / AI Overview
```

For the first 30-day proof of concept, deploy the complete Docker Compose stack
to one Linux VM in Australia East (Azure) or Sydney (Vultr PoC — see
[`docs/deployment/geomode-vultr.md`](../deployment/geomode-vultr.md)):

- VM: `Standard_B2ms` (2 vCPU, 8 GiB)
- OS: Ubuntu 24.04 LTS
- Disk: 64 GiB Standard SSD
- Containers: Elmo web, Elmo worker, migration job, PostgreSQL
- Browser concurrency: `1`
- Public ingress: HTTPS only, through Caddy or nginx
- PostgreSQL: private Docker network only; never publish port `5432`

This keeps the deployment understandable and leaves enough memory for one
Chromium session. Move PostgreSQL to a managed service and browser workers to
Azure Container Apps Jobs only after the capture workload is measured.

## Geomode Fork Changes

Elmo already dispatches providers through `SCRAPE_TARGETS` and the
`Provider.run()` contract. The `sodown4thecause/geomode` fork should make these
scoped changes:

1. Add `packages/lib/src/providers/registry/cloakbrowser.ts`.
2. Register it in `packages/lib/src/providers/index.ts`.
3. Add `CLOAKBROWSER_*` variables to `packages/config/src/env.ts`.
4. Add `cloakbrowser` and `playwright-core` to `packages/lib/package.json`.
5. Extend the worker Docker target from `cloakhq/cloakbrowser`, or install the
   same Linux dependencies and pre-download its Chromium binary.
6. Add provider unit tests and one opt-in live smoke test.
7. Remove Bright Data from the recommended CLI setup in the fork; do not delete
   the upstream provider unless maintaining it creates a real burden.

Target configuration:

```env
SCRAPE_TARGETS=chatgpt:cloakbrowser:online,perplexity:cloakbrowser:online,google-ai-mode:cloakbrowser:online
CLOAKBROWSER_HEADLESS=false
CLOAKBROWSER_HUMANIZE=true
CLOAKBROWSER_CONCURRENCY=1
CLOAKBROWSER_TIMEOUT_MS=180000
CLOAKBROWSER_PROXY_URL=${LIGHTNING_PROXY_URL}
```

`LIGHTNING_PROXY_URL` is the complete HTTP or SOCKS5 URL generated by the
Lightning Proxies dashboard, including gateway, port, username, and password.
Do not pass the Lightning account/API key to CloakBrowser as though it were a
proxy URL.

The provider must return Elmo's normal `ScrapeResult`:

```ts
{
  textContent: string
  rawOutput: unknown
  webQueries: string[]
  citations: Citation[]
  modelVersion?: string
}
```

Do not make DOM selectors the only extraction path. Prefer, in order:

1. Structured page state or accessible rendered content.
2. Stable semantic landmarks and labels.
3. Network responses when their use is permitted and stable.
4. Engine-specific CSS selectors as a versioned fallback.

Each adapter should save a small diagnostic artifact on failure: URL, engine,
timestamp, sanitized HTML excerpt, and screenshot path. Never persist cookies,
tokens, prompts from other tenants, or proxy credentials in diagnostics.

## Session Strategy

Use one persistent browser profile per engine and region, stored on an encrypted
VM data volume. Profiles reduce repeated login and consent flows.

- Separate ChatGPT, Perplexity, and Google profiles.
- Never share a profile across customer tenants if the target session contains
  customer-specific account data.
- Serialize work per profile with a database or pg-boss lock.
- Recycle the browser after a bounded number of prompts to control memory.
- Detect login, consent, challenge, and rate-limit pages explicitly.
- Return a typed provider error instead of treating a challenge page as an
  answer.

## FlowIntent Integration

The current `lib/geo/oneglanse-client.ts` is a temporary provider facade.
Replace browser-backed branches with `lib/geo/elmo-client.ts` once the fork
has a stable API (`POST /api/v1/runs`, `GET /api/v1/runs/:id`). Enable with
`GEO_ELMO_ENABLED=true`, `ELMO_API_URL`, and `ELMO_API_KEY`.

Keep these contracts in FlowIntent:

- `GeoEngineResult` remains the normalized result type.
- `geo_runs` remains the user-facing run history.
- FlowIntent owns brand, competitor, subscription, and tenant authorization.
- Elmo owns prompt scheduling, raw captures, citations, and provider health.
- Raw browser output stays in Azure with a short retention window.
- FlowIntent stores normalized response text, citations, scores, and an Elmo run
  identifier.

Do not call CloakBrowser directly from `/api/geo/runs`. A browser run can exceed
Vercel request limits and needs queueing, retry, cancellation, and concurrency
control.

## Lightning Residential Proxy

CloakBrowser changes browser fingerprints; it does not provide residential IPs
or proxy rotation. Use Lightning Proxies as the residential egress provider for
browser-backed GEO checks.

Lightning supports HTTP/SOCKS5, user/password or IP allowlisting, geo-targeting,
and sticky or rotating sessions. Generate a proxy endpoint in its dashboard and
store the complete URL as the `LIGHTNING_PROXY_URL` secret. Prefer sticky
sessions for logged-in ChatGPT and Perplexity profiles so the exit IP does not
change during a browser session. Rotating sessions are suitable for independent
Google queries.

The Lightning account/API key is a management credential and must be stored
separately as `LIGHTNING_PROXY_API_KEY` only if the deployment later calls the
provider's management API. The browser worker should normally need only the
generated `LIGHTNING_PROXY_URL`.

Proxy charges are external to Azure and will not consume the Azure credit.

## Security

- Store Elmo API credentials, `LIGHTNING_PROXY_URL`, any
  `LIGHTNING_PROXY_API_KEY`, and session secrets in Azure Key Vault or
  root-readable Docker secrets.
- Put the Elmo API behind HTTPS and an allowlist or private tunnel.
- Use a dedicated non-root service account for the stack.
- Do not expose CloakBrowser CDP port `9222`; CDP grants complete browser control.
- Do not expose PostgreSQL publicly.
- Encrypt persistent profiles and database backups.
- Redact cookies, authorization headers, and account identifiers from logs.
- Pin CloakBrowser and Elmo image versions; do not deploy floating `latest`
  tags to production.

## Rollout

1. Add the `cloakbrowser` provider to `sodown4thecause/geomode`.
2. Prove one unauthenticated Google AI surface capture locally.
3. Prove ChatGPT and Perplexity with dedicated test accounts and persistent
   profiles.
4. Deploy the single-VM Docker stack to Azure.
5. Add a narrow authenticated run/status API to `sodown4thecause/geomode`.
6. Add the FlowIntent Elmo client and switch `/api/geo/runs` behind a feature
   flag.
7. Run a 20-prompt engine matrix and measure success rate, latency, memory, and
   challenge frequency.
8. Enable scheduled GEO checks only after interactive runs are reliable.

## Acceptance Criteria

- `chatgpt:cloakbrowser:online`, `perplexity:cloakbrowser:online`, and
  `google-ai-mode:cloakbrowser:online` pass Elmo target validation.
- One browser process runs at a time on the B2ms VM.
- Every successful run returns non-empty answer text or an explicit
  `no_answer` status.
- Citations are normalized and deduplicated.
- Challenge/login/rate-limit pages are classified, not stored as answers.
- A browser crash is retried once with a fresh process.
- FlowIntent can create a run, poll status, and persist the normalized result.
- No browser, CDP, database, or secret-management port is publicly exposed.
